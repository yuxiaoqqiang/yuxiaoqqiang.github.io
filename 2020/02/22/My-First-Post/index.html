使用kubeadm部署kubernetes集群

1.准备工作

1.1 环境信息

OS : Ubuntu 18.04 LTS

机器名称 ip地址
master 192.168.105.214
node1 192.168.105.215
node2 192.168.105.216
node3 192.168.105.217
node4 192.168.105.218
node5 192.168.105.219

这里注意，如果是申请的虚拟机，请一定注意机器名称，申请的虚拟机默认的名称都是一样的，需要修改成不同的名称，否则在最后其他节点加入集群时会出现问题。

注意，部署Kubernetes集群需要root权限

1.2 关闭防火墙

关闭ufw防火墙，Ubuntu默认未开启，若开启将其关闭

备注：ufw的底层是使用iptables进行数据过滤，建立在iptables之上，这可能会与Docker产生冲突

1.3 禁用SELINUX

Ubuntu默认不安装selinux，若安装了的话，采用如下步骤禁用

1.4 开启内核ipv4转发

修改/etc/sysctl.conf，开启ipv4转发

1.5 禁用swap分区

还需修改/etc/fstab文件，注释掉swap的自动挂载，防止重启后swap启用。

1.6 配置iptables参数

配置iptables参数，使得流经网桥的流量也经过iptables/netfilter防火墙

1.7 配置免密登录

2.8 docker启动参数配置

为docker做如下配置：

设置docker的cgroup driver为systemd，与kubelet保持一致。

设置阿里云镜像库加速dockerhub的镜像。国内访问dockerhub不稳定，将对dockerhub的镜像拉取代理到阿里云镜像库(xxxxx替换为阿里云生成的镜像代理仓库地址)

禁用iptables

如果想让podIP可路由的话，设置docker不再对podIP做MASQUERADE，否则docker会将podIP这个源地址SNAT成nodeIP

设置docker存储驱动为overlay2（需要linux kernel版本在4.0以上，docker版本大于1.12）

根据业务规划修改容器实例存储根路径（默认路径是/var/lib/docker）
